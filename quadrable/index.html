## Partial Trees

* After proof processing, we are left with a *partial tree* * Not a complete tree because it contains witness nodes (red) * We can perform lookups, insertions, modifications, and deletes, as long as we never traverse to a red node

## Witness Nodes

* Witness nodes can be "summarising" very large sub-trees * This is why proofs can be small, even when the database is very large * When new proofs are received, we can replace witnesses with the new nodes, expanding the partial tree

## Non-Inclusion

* How do you prove a key is *not* in the tree? * Proof creator just needs to make it so negative lookups don't encounter witness nodes * Two methods for negative lookups: Either an empty branch or a leaf blocking path to queried key
